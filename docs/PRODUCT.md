# Product

## Users

IrisAI is for Brawl Stars players running a local automation session from a Mac.
They keep the dashboard open while they work or stay nearby, returning to it to
confirm that the bot, emulator, current brawler, and selected playstyle are
behaving as expected. Their main job is to start, monitor, pause, and adjust a
session without reading technical logs.

## Product Purpose

IrisAI makes local Brawl Stars automation observable and controllable. A
successful interface states what the bot is doing, which brawler it is using,
which playstyle is active, and what recent matches produced. Normal operation
should feel quiet and dependable, while detailed diagnostics remain available
when a real problem occurs.

## Brand Personality

Calm, capable, and practical. IrisAI feels like a well-kept macOS tool and a
tactical runbook, not an AI demonstration or a game overlay. The voice is
concise, direct, and international English.

## Anti-References

Avoid cyberpunk control rooms, neon gaming dashboards, decorative gradients,
oversized hero metrics, AI-chat visual language, noisy log walls, and generic
card grids. The interface must not depend on color alone to communicate state.

## Design Principles

1. State before spectacle: show whether the bot is healthy and what it is doing before secondary controls.
2. The current run has a home: brawler, trophies, streak, playstyle, and latest result belong together.
3. Results stay close to action: recent matches and recent events are concise, legible, and nearby.
4. Quiet by default: normal operation exposes meaningful events, while technical detail stays behind an explicit debug control.
5. Familiar tools build trust: use macOS-like controls and a Linear, Notion, and Codex level of information discipline.

## Accessibility

Meet WCAG AA contrast in both themes. Support keyboard navigation, visible focus
states, reduced motion, responsive layouts from narrow mobile screens through
large desktop displays, and text labels alongside status color.
